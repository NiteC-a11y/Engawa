## ソースコードレビュー

### Findings

1. **ACP プロセス終了時に pending request が解放されず、initialize/prompt が永久待ちになる可能性がある**
   - 参照: `acp.py:114-119`, `acp.py:129-133`, `acp.py:176-199`, `acp.py:218-232`
   - `ACPClient.request()` は future を `_pending` に積んで応答待ちするが、`reader()` は stdout EOF で単に break するだけで、未解決 future に例外を入れない。adapter が起動直後または prompt 中に落ちると、`spawn()` や `prompt()` が戻らない。
   - 影響: 認証失敗、npx/adapter クラッシュ、ネットワーク断などでアプリ全体がハングし得る。
   - 推奨: `reader()` の終了時に `_pending` の future 全てへ `ConnectionError` 等を set_exception する。必要なら `request()` に timeout を持たせる。

2. **Web UI の shutdown がスレッド境界を跨いで `window.destroy()` を呼ぶ**
   - 参照: `engawa_main.py:73-91`, `views.py:262-267`
   - `run_web()` の background thread の `finally` から `window.destroy()` を呼んでいる。pywebview の GUI 操作は通常メイン/UI スレッド制約があるため、環境によっては不安定になる可能性がある。
   - 影響: `/quit` や scheduler 終了時の窓閉じで例外または閉じ残りが出る恐れ。
   - 推奨: 現時点では低優先。問題が再現したら、pywebview が保証する API 経路に寄せる、またはメインスレッド側に close signal を渡して閉じる。少なくとも shutdown smoke を追加する。

3. **テストファイルがリポジトリに残っていない**
   - 参照: `Backlog.md:15`, `Backlog.md:25-29`, `Backlog.md:39`, `Backlog.md:48`
   - Backlog では多数の PASS が記録されているが、`rg --files C:\dev\Engawa` 上ではテストファイルが見当たらない。
   - 影響: 今後の変更で cancel、guest、WebView poll、topic whitelist などの重要挙動を回帰検知できない。
   - 推奨: 少なくとも pure Python で可能な `collapse_ws`, `corner_xy`, topic parsing/whitelist, `Scheduler` + `CaptureView` の fake agent テストをリポジトリに戻す。

4. **設定の範囲検証が弱く、壊れた値が極端な挙動に直結する**
   - 参照: `config.py:39-51`, `scheduler.py:18-24`, `sources.py:28-35`
   - cast はしているが、範囲は見ていない。例: 負の `ENGAWA_TOPIC_REFRESH_MIN` で毎 tick 取得、`ENGAWA_GUEST_PROB > 1` で常時来訪になる。
   - 影響: 設定ミスが静かに不自然な常駐挙動や外部取得頻度の増加になる。
   - 推奨: 防御的改善として clamp または validation を追加し、無効値は既定へ戻す。少なくとも確率は 0..1、間隔は正数にする。緊急度は低い。

5. **`session/request_permission` の応答 JSON は実装を正として文書に明記した方がよい**
   - 参照: `TECH_RULES.md:42`, `engawa-acp-spec.md:143`, `acp.py:159-160`
   - 現行実装は `{"outcome": {"outcome": "cancelled"}}` を返す。これは ACP の `RequestPermissionResponse` として妥当そうに見える。旧仕様の selected/deny は旧構想文書なので、現行実装の問題とは扱わない。
   - 影響: 実害は薄い。読み手向けの明文化不足。
   - 推奨: `TECH_RULES.md` に実際の応答 JSON を追記する程度でよい。

### 良い点

- `EventSource` / `Scheduler` / `View` / `AcpAgent` の境界は明確で、ADR-0013 の狙いと実装が概ね一致している。
