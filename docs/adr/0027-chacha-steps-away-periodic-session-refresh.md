# ADR-0027: 茶々の「中座」で長命セッションを定期リフレッシュ（世界観に溶かした劣化根治）

- ステータス: Accepted（実装済み・2026-07-03）
- 日付: 2026-07-03
- 関連: ADR-0005（住人のセッションは長命）, ADR-0013（イベント源/スケジューラ）, ADR-0011（イベント=アーク）, ADR-0025（有界代打・減衰テンポ）, ADR-0002（サブスク認証）, 染み出し Open Question（Backlog）＋ガード commit `9d3a297`
- 影響: ADR-0005 を **refine（追補）**。「セッションは長命」は保つが、*長命ゆえに溜まる劣化*で破綻しないよう **定期的に中座で若返らせる**。染み出し不具合の**根治**（ガード `strip_resident_leak` は対症＝表示から隠すだけ）。

## 背景 / 課題
長時間稼働すると茶々ソロ出力に「注入プロンプトの復唱＋地の思考(英語/メタ)＋本物の台詞」が混じる（7/3 実機報告）。内部 compaction で「茶々として答えるだけ」の枠を失う筋＝**長命セッション（ADR-0005）の劣化**。commit `9d3a297` の表示ガード＋`/restart` は:
- ガード＝崩れた出力を**画面から隠す対症**（1回崩れたターンは出てから消えるわけでなく、そもそもモデルが崩れている）。
- `/restart`＝**手動レバー**（ユーザーが気づいて叩く必要がある）。

根治には「**劣化する前にセッションを若返らせる**」自動化が要る。だが冷たい auto-restart（無演出でセッションを捨てる）は、環境反応型の住人という世界観（茶々は道具でなく縁側の住人・原則#2）を壊す。ユーザー着想: **「茶々をたまに不在にして、その裏でリセットすればいい。茶々もトイレくらい行くしね」**（7/3）。＝技術的なセッション更新を**茶々の生活動作に溶かす**。忘却すら「席を外してたら話の続きを忘れた」で自然になる。

## 決定
**茶々が「たまに中座」し、その不在の裏で住人セッションを黙って張り直す（`_restart_resident()` を流用）。** 中座の断り／戻りの台詞は**ローカル定型**（LLM 非経由＝劣化しているかもしれない今のセッションに喋らせない・`guest_timeout_leave` と同じ流儀）。

- **シーケンス**: 茶々が短く断る（例「ちょっと手ぇ洗てくるわ」）→ 少し不在（つぶやき抑制・config 秒）→ 裏で `_restart_resident()`（新セッション）→ 戻る（例「ふぅ、すっきりした」）→ 以後まっさら。
- **idle 限定**: 喋っている最中／人待ち／来訪(room)中／対局(game)中は行かない＝会話・場を切らない（トイレは間が空いた時に立つ）。busy なら「圧が満ちて」いても次の idle まで**遅延**。
- **「たまに」を根治に効かせる＝ただのランダムにしない**: 前回リフレッシュからの経過（住人ターン数 or 分）で**中座確率が上がり、上限で必ず発火**。＝「たまに」の体裁のまま、一定周期でセッションが必ず若返る（劣化を追い越す）。ADR-0025 の減衰テンポの逆で“圧が溜まる”モデル。
- **座組へのはめ方**: 中座は **EventSource** としてレジストリに挿す（`eligible`＝圧＋idle ゲート／`cooldown`＝中座の最短間隔・ADR-0013 の event-source/scheduler に素直）。scheduler は「この源が発火したら leave→(不在gap)→reset→return を回す」特別扱いを1点持つ（セッション lifecycle は scheduler の責務）。
- **文脈は毎回まるっと忘れる**: idle 限定ゆえ*話の途中で忘れる*ことは無い＝トイレ枠で違和感なし。要約引き継ぎ（前の話を新セッションへ）は劣化テキスト再注入リスク＋複雑ゆえ**今は入れない**（将来オプション）。

## なぜ原則を壊さないか
- **原則#2（住人らしさ）を守る方向**: セッション更新という技術操作を「トイレ/席立ち」という生活動作に溶かす＝むしろ住人らしさを強める。冷たい restart の逆。
- **原則#3（無際限の自律往復禁止）とは無関係**: これは住人1人の生活動作であって AI↔AI 会話ではない。中座中は AI を動かさない（むしろ黙る）。
- **ADR-0005（長命）を保守**: 「長命」を捨てるのでなく、*劣化で破綻させない*ための定期メンテ。一度の中座で文脈は切れるが、1回の sitting 内では滅多に起きない頻度に調整＝地続き感は保つ。

## 検討した代替案
- **冷たい auto-restart（無演出でセッションを捨てる）**: 世界観を壊す（原則#2）。却下＝中座に溶かす。
- **Nターン固定で必ずリセット（演出なし）**: 硬く不自然。中座の“たまに＋圧で必ず”の方が生き物らしく、かつ周期保証も満たす。却下。
- **ガード（`strip_resident_leak`）だけで済ませる**: 対症のみ＝モデル出力は崩れたまま／根治しない。併用はするが単独では不足。却下。
- **文脈を要約して新セッションへ引き継ぐ**: 劣化テキストを新セッションに再注入するリスク＋複雑。今は却下（将来オプション＝「戻ったら少しだけ前の話を覚えてる」）。
- **`/restart`（手動）のみ**: ユーザーが崩れに気づいて叩く前提＝根治でない。手動レバーとしては残す。

## 影響 / 帰結
- **新 EventSource「中座」**（`sources.py` + registry）: `eligible`＝圧（前回 refresh からの経過）＋idle ゲート、`cooldown`＝最短間隔。
- **`prompts.py`**: 中座の leave/return ローカル定型（数バリアント・LLM 非経由）。
- **`scheduler.py`**: 中座源の発火で leave→不在gap→`_restart_resident()`→return を回す。refresh 起点（ターン数/時刻）を保持。
- **config**: 圧の閾値（soft/hard）・cooldown・不在秒・`0`で無効（`ENGAWA_ABSENCE_*` 等）。ADR-0002 の枠内（認証・課金に無関係）。
- **テスト**: `eligible` の圧/idle ゲート・発火で `_restart_resident` が呼ばれる・busy 中は行かない・leave/return が出る（fake resident/spawn・実 LLM 不要）。実発話トーンは目視。
- **presentation（後日・P5）**: 不在中は「空っぽの縁側」スプライト（茶々居ない）＝ADR-0019 の意味 state 拡張。今回はスコープ外。

## 備考（Open Questions）
- 中座頻度の体感バランスは `engawa.json` で調整（既定は「活動 N ターン/分で圧が満ちる→次の idle」）。
- room/game 中の劣化は中座では触れない（idle 限定）＝別途 room 側ガード横展開は Backlog。
- 「戻ったら少しだけ前の話を覚えてる」軽い引き継ぎは将来オプション（劣化再注入に注意）。
