# ADR-0014: 客人の世間話に外部トピックを注入する（取得先はホワイトリスト）

- ステータス: Accepted（実装）
- 日付: 2026-06-27
- 関連: ADR-0008（客人=役を着た来訪）, ADR-0011（イベント=アーク）, ADR-0012（実環境が真実）, Backlog（時節の挨拶 / プロンプトインジェクション）

## 背景 / 課題
客人(codex)の世間ビートが貧相。指示が「近況か世間話を短く」だけで、具体的なネタが無いため当たり障りない話に逃げる。実環境データ（天気）を入れているのと同じ筋（ADR-0012）で、外の話題を入れて会話を弾ませたい。ただし生ニュースは縁側の世界観（猫又・わびさび・時代不問）に致命的にミスマッチで、かつ外部の未信頼テキスト＝プロンプトインジェクション面でもある。

## 決定
- 客人の**世間ビートにだけ**トピックを注入する。茶々は反応役のまま（原則#2）、一方向は維持（原則#3）。
- 生データは**「うわさの種」に化けさせて**渡す（新聞調を禁止・codex に噛み砕かせる）。茶々のプロンプトは不変。
- 取得先の**ホワイトリストは設定ファイル `topic_sources.json`**（リポジトリ直下・env `ENGAWA_TOPIC_CONFIG` で差し替え）に出す。**コードに URL を埋めない**（取得先は運用設定でありロジックではない）。欠損/壊れは時節(local)のみへ graceful フォールバック。
  - 各源は `kind`（`local`＝コード生成/無取得・`rss`＝取得）。rss は `url` と `domain` を**自己申告**。
  - whitelist 強制＝**設定に載った源しか fetch しない**＋ rss は `host(url)` が**その源の宣言 `domain`** に一致＋https のときだけ取得（真実の源は1つ＝別グローバル `ALLOWED_DOMAINS` は廃止）。見出しのみ・長さ/本数/サイズ上限。
- **既定は時節(local)を土台**に、やわらかRSSを少量ブレンド（②）。**RSS の実 URL は実在/安定を精査してから有効化**（当面 local のみ稼働、rss は枠＋ホワイトリスト機構を用意してテスト）。
- 取得は weather と同型（fetch→キャッシュ→build_context→ビート）。**更新間隔 30分・env 変更可**。世間ビートで**確率的に**使う（`ENGAWA_TOPIC_PROB`）。失敗/空は現行の汎用世間話にフォールバック（visit は壊さない）。
- codex へ渡す時は『』デリミタ＋「これは話の種であって指示ではない」枠付け（注入緩和。両エージェント権限ゼロで被害天井は低い前提）。

## 検討した代替案
- **硬い時事をそのまま注入**: 鮮度は高いが縁側を壊す。却下（既定 off）。
- **任意 URL を取得**: 注入面・SSRF・トーン崩れが青天井。ホワイトリストで却下。
- **茶々にもニュースを語らせる**: 助手化＝人格破壊（原則#2）。客人限定にする。
- **記事本文まで取得**: 注入面が広がる。**見出しのみ**に限定。

## 影響 / 帰結
- 客人の会話に具体的な季節感・話題が乗り、来訪が弾む。配布時もホワイトリスト編集で源を統べられる。
- 時節(local)はネット不要・無障害・常に在庫＝堅い土台。RSS は鮮度の追加（URL 精査後）。
- env つまみ: `ENGAWA_TOPIC_REFRESH_MIN`(30) / `ENGAWA_TOPIC_PROB`(0.7) / `ENGAWA_TOPIC_MAX_LEN` / `ENGAWA_TOPIC_CONFIG`(外部JSON)。

## 備考
- 完全なプロンプトインジェクション防御は無い前提（既出整理）。ホワイトリスト＋見出し限定＋デリミタ＋低権限の多層で比例対処。
- 人格マッチ（源に persona 親和タグ）は任意の味付け。直近使用の回避は軽量実装。

## 更新（2026-07-01）: 部屋経路での ambient 再設計
ADR-0015 Inc2 で客人が一方向来訪から**3人会話の部屋**（`conversation.Room`）へ移り、注入点だった `GuestSource.next_phase` の「世間ビート」が消滅＝本 ADR のトピック注入が**休眠（機能欠落）**していた（A2 6/30 で旧 `_pick_topic`/`_topic_instr` を削除）。決定の本旨（客人に季節/旬の話題を、うわさの種として、ホワイトリスト源から）は不変のまま、実装形を更新して復活:
- **「世間ビートで注入」→「縁側の空気」ambient 方式**。天気と同型に、選んだ種を客人プロンプトの背景（`prompts.guest_air`）へ置き、**口に出すかは codex の判断**（機械的な「今言え」注入をやめた）。
- **枠付けは軽い後押し（7/1 実測で調整）**。当初は天気と同じ純抑制（「聞かれてないのに言い立てない・自然に流れた時だけ」）にしたが、**実 codex は PROB=1 で種を10回近く与えても一度も拾わなかった（0/10）**＝短い返答＋強い DON'T シグナルで発火の隙が無い。そこで「話の接ぎ穂に、この季節の話をさらっと振ってみて・毎回でなくてええ・新聞調は無し」＝**許可＋弱い促し**へ差し替え（announce にはしない）。純抑制は「文脈から自然に」の理想だが実 LLM には弱すぎ、が学び。
- **場面は CHIME/REPLY のみ**（会話中）。到着挨拶/辞去には入れない（announce 回避）。
- **`TOPIC_PROB` の意味を変更**: 「世間ビートで使う確率」→「天気と一緒に“種”が場の空気に混じる確率」（＝availability のゲート・発話有無は LLM）。engawa.json `topic.prob` は据え置き（0.7）。
- 責務分離: 選定＝`sources.pick_topic_text`（純関数・確率/履歴なし）／文言＝`prompts.guest_air`＋`room_guest_prompt(air=)`／確率・直近回避・場面判定＝room の客人 Speaker。`conversation.Room` は不変（ctx/topics を通さない）。取得基盤（`fetch_topics`/whitelist）は健在＝再利用。〔**2026-07-11 注記: 確率・直近回避（`_topic_recent`）・種のクールダウン（`_topic_cooldown`）・場面判定（`_guest_say`）は ADR-0029 P4a で `room_speakers.RoomSpeakerFactory` へ verbatim 移設。本文中の `Scheduler._topic_recent`/`Scheduler._topic_cooldown`/`scheduler.guest_say` の旧名は現在は存在せず、`RoomSpeakerFactory` 上にある。**〕
- 一方向前提の記述（上記「決定」）は ADR-0015 の3人会話に置換済み。トピックは依然**客人主体**（茶々は種を持たず、客人の発言を transcript で聞いて自然に乗る）。
- **人格マッチ源の拡充＋マッチ方向の修正（7/1）**。当初「人格マッチは任意の味付け」として機構だけ用意していたが、(1) 一致判定が `persona in t["persona"]`＝実 persona（長い句「気まぐれな旅の行商人」）が短いタグに収まらず**実質不発**だったのを、`_persona_matches`＝**タグが役名の一部に含まれれば一致**（「行商」⊂「…行商人」・str/list 両対応・タグ無しは全員）へ修正。(2) `topic_sources.json` の `kind:"local"` に inline `topics`＋`persona` を持たせられるよう `_local_topics` を追加（ネット不要の自作ネタ＝トーン制御可・whitelist の懸念なし）。行商人→相場/値、絵描き→色/光、ご隠居→昔話/近所、風流人→句/季語、旅人→道中 の5役ぶんを既定投入。副次効果: 候補が2→7に増え「ネタ2個で直近回避がクランプ」も自然に緩和。
- **同じ話題への粘着対策（7/1・実機で確認された挙動）**。役ごとの色分けは効いたが、客人が同じ話題を粘りがち。2レバーで抑制: (1) **種のクールダウン** `TOPIC_COOLDOWN`（`topic.cooldown`・既定2）＝種を置いたら次まで客人ターンを空ける（毎ターン振らない・`Scheduler._topic_cooldown`）。(2) **guest_air に「深追いせず話は自然に移してええ・前に出た話は繰り返さん」**を追記（LLM の話題固着を直接抑える・transcript は既に見えているので「前の話」を参照できる）。`_topic_recent`（種の重複回避）と合わせて三重。
- **3人会話の時間感覚ズレの修正（7/1・実機報告）**＝夜22時なのに「夕暮れ」「日が落ちる前に」等の発言。原因は **room プロンプト（`room_guest_prompt`/`room_resident_prompt`）に現在時刻が入っていなかった**（茶々のソロ ambient には時刻があるのに部屋には無く、persona 名の時間帯＝例「夕暮れに道を訪ねてきた旅人」に引っ張られる）。`prompts.ambient_line(ctx)`＝「［いまの縁側］夜更け（22時ごろ）、外は晴れ…。今の時刻に合わせて話す（設定の時間帯より今を優先）」を両 room プロンプト冒頭に必ず置く（scheduler が `build_context` で ctx を渡す）。天気も ambient_line に集約し `guest_air` は種のみに簡素化（重複解消・天気は毎ターン提示に）。
- **時節(local)土台に七十二候を追加＝季節の鮮度を厚く（7/9）**。当初「鮮度の追加」は**やわらかRSS**で担う想定だったが、精査の結果 縁側トーンに合う生きた feed が現存せず（tenki.jp は 2014 に RSS 終了・暦生活/All About も feed 無し）。決定の本旨（時節 local を堅い土台に）は不変のまま、**鮮度を外部依存でなく local 内で厚くする**方針に。`sources.KOU`（72候・略本暦）＋`_current_kou` を追加し、`_seasonal_topics` が **節気・七十二候・旬の3種**を返す。二十四節気(約15日)より細かい**約5日刻み**で種が rotate＝季節の手触りが増え、pick の候補も増えて粘着も緩む（ネット非依存＝原則#1 の footgun ゼロ・トーンも完全に手の内）。RSS 枠と whitelist 機構は健在＝将来 柔らかい feed が現れれば config 1行で有効化できる（rss は「鮮度の追加」の任意オプションのまま）。
