# ADR-0020: モデルは config で選ぶ（住人=ANTHROPIC_MODEL / 客人=CODEX_CONFIG・未指定はアダプタ既定）

- ステータス: Accepted（実装 2026-06-28）
- 日付: 2026-06-28
- 関連: ADR-0002（サブスク認証・APIキーは使わない）, ADR-0001（ACP）, ADR-0008（客人=codex）

## 背景 / 課題
住人（茶々=Claude）と客人（codex）に使うモデルを切り替えたい（常用 opus／実験 sonnet 等）。ただし ADR-0002 でサブスク認証＋API キー除去を守っており、API を直叩きしてモデルを渡す経路は無い。アダプタ（claude-code-acp / codex-acp）を経由したまま、サブスク認証を壊さずモデルだけ指定する手段が要る。

## 決定
- **住人（Claude）**: 子プロセス env の `ANTHROPIC_MODEL` に載せる（Claude Code が尊重・`opus` / `claude-opus-4-8` / `opus[1m]` 等）。
- **客人（codex）**: `CODEX_CONFIG` に `{"model": …}` を JSON で載せる（codex-acp が Codex セッション設定へマージ）。
- つまみは config 主導: `ENGAWA_MODEL` / `ENGAWA_CODEX_MODEL`（`engawa.json` の `model.{resident,guest}` 可・**env 優先**・原則4）。
- **未指定（空）はアダプタ既定のまま**＝注入しない（現状の挙動を変えない）。サブスク認証でも有効。API キーは載せない（ADR-0002 と分離）。
- `/model` で現在のモデルを表示（アダプタが session/new で報告した実モデルを優先、無ければ要求値、どちらも無ければ「不明」）。**縁側への操作なので茶々には流さない**（ADR-0007）。

## 検討した代替案
- **API 直叩きでモデル指定**: ADR-0002（サブスク・課金事故防止）に反する。却下。
- **コードにモデルをハードコード**: 端末ごとに変えられない。config 主導（env > json > 既定）にする。
- **未指定時に既定モデルを明示注入**: アダプタの将来の既定変更を上書きしてしまう。**未指定はアダプタ既定のまま**にして現状維持。

## 影響 / 帰結
- 端末ごと・実験ごとにモデルを env / `engawa.json` で切替可。サブスク枠を意識して opus を常用しつつ ambient を間引く運用ができる（tick/ mutter 設定と併用）。
- `acp.py` の `_model_env` / `_child_env`、起動行の `茶々=<model>` 表示。ユニットで env 組み立てを担保。
- codex 側 `CODEX_CONFIG` 経由の実効はアダプタ実装依存（無指定なら無害）。実モデル切替の実機確認はユーザー（GUI / 実アダプタ）。

## 備考
- 「アダプタが報告する実モデル」を真実とし、こちらの要求値は補助表示に留める（版差・別名を取り違えない）。
