# ADR-0026: LLM 接続は Agent ポートで中立化（ACP と API の2アダプタ・MCP は直交レイヤ）

- ステータス: Proposed（構想・ADR-0001 の refine。用語と境界を確定・実装は段階的で未着手）
- 日付: 2026-07-03
- 関連: ADR-0001（ローカル駆動は MCP でなく ACP）, ADR-0002（サブスク認証・APIキー不使用）, ADR-0006（cancel 優先）, ADR-0008（客人＝ツールを持たせない）, ADR-0020（モデルは config で選ぶ）, ADR-0021（ACP timeout/段階回復）
- 影響: ADR-0001 を **refine（追補）**。Supersede しない——「ローカル駆動の既定は ACP」は不変。その口を中立ポートの背後に置き、非ACPバックエンド（ローカル API）を将来許容する余地を、境界と用語ごと明文化する。

## 背景 / 課題
ローカル LLM（例: LM Studio）を茶々/客人のバックエンドに試したい、という話から「LLM に接続するアダプタは **ACP / MCP / API** の3種では？」という整理案が出た。しかしこれを対等に並べると**層が混ざる**：ACP と API は「モデルの推論に届く経路」だが、**MCP は到達経路ではなく「届いたモデルに道具/データを足す」直交レイヤ**である。現に Engawa は既に `session/new` で `mcpServers: []` を渡しており（`src/acp.py:304`）、MCP は *ACP エージェントに渡すパラメータ* として扱っている＝経路の兄弟ではない。

また ADR-0015/0025 で会話は `Speaker`（Strategy/DI）注入によりドメイン（scheduler/conversation）から切り離され、接続層 `AcpAgent` がドメインに見せる面は実質 `prompt / cancel / close ＋ 人格つきファクトリ` の3+2に収束している。つまり**ポートの線はほぼ引かれている**が、`AcpAgent` が「ポート契約」と「ACP 実装」を1クラスで兼ねているため、非ACPバックエンドを足す口が言語化されていない。

## 決定
**「LLM の推論に届く経路」を `Agent` ポートで中立化し、そのアダプタは2系統（ACP / API）に限定する。MCP はポートのアダプタではなく、アダプタの中/上に載る直交レイヤと定義する。**

### 2軸マップ（この ADR の地図）
```
軸A ＝ どうモデルの推論に“届く”か  … これが Agent ポートのアダプタ
   ├─ ACP  … ローカルの“エージェント”を JSON-RPC/stdio で駆動（Claude Code / Codex）。
   │          認証もツールループもエージェント持ち。← 現行・既定（ADR-0001）
   └─ API  … モデルの補完エンドポイントを直に叩く（HTTP）。人格/stream/cancel は自前。
              └─ ローカル OpenAI 互換：LM Studio(:1234/v1) / Ollama / llama.cpp / vLLM
                 （クラウド従量 API も技術的にはここだが ADR-0002 の課金NGにより原則外＝ローカル/自ホスト限定）

軸B ＝ 届いたモデルに“道具/データを足す”か  … 推論の到達手段ではない（直交）
   └─ MCP  … サーバがツール/リソースを公開、エージェントが消費。
              ACP なら session/new の mcpServers に列挙。API アダプタなら自前のツールループに載せる。
```

### 決定の骨子
1. **`Agent` ポート（プロバイダ中立）を抽出**する。契約はドメインの必要から定義：
   - `prompt(text, on_chunk) -> stopReason`（**streaming 必須**）
   - `cancel()`（**必須**・ADR-0006 の cancel 優先を担う load-bearing）
   - `close()`
   - 生成ファクトリ（住人/客人＝**人格を引数で明示的に受け取る**）
2. **アダプタは ACP と API の2系統のみ**。`AcpAgent` は「ポートの1実装」に降格（`ACPClient` の JSON-RPC トランスポートは `AcpAgent` 内部に留置）。`OpenAIAgent`（ローカル OpenAI 互換）を将来のもう1実装として同じ面に並べる。
3. **MCP は Agent ポートのアダプタに数えない**。ツール供給の直交レイヤとして、必要時に (a) ACP の `mcpServers`、(b) API アダプタ内のツールループ、のどちらかに載せる。MCP の推論経路 `sampling/createMessage` は**逆トポロジ＋ deprecated（SEP-2577・ADR-0001）**ゆえ到達手段に採らない。
4. **人格注入を裏口から契約へ昇格**。現状「住人＝cwd の CLAUDE.md をアダプタが読む」（ADR-0003）は ACP 依存の側チャネル。中立ポートでは **system/persona プロンプトを引数で渡す**形にし、`AcpAgent` 側だけ「cwd に CLAUDE.md を置く」へ変換する。
5. **実装は段階的（YAGNI）**。まず「ポート抽出のみ・実装は ACP のまま・振る舞い不変」を行い、第2実装（`OpenAIAgent`）を入れる段になって本 ADR を Accepted に更新する。第2実装が入るまで ACP を唯一のアダプタに保つ。

## 検討した代替案
- **ACP / MCP / API を対等な3アダプタにする（当初案）**: MCP は到達経路でなく直交レイヤ＝層が混ざる。却下し、2軸に分離（本決定）。
- **ポート化せず「ローカルは ACP シムだけ」で通す**: LM Studio を ACP を喋る薄いシムで包み `ENGAWA_ACP_CMD` に差す案。動くし有効な選択肢なので**ポートの1アダプタとして残す**が、これ単独だとローカルでも常に npx/子プロセス/JSON-RPC を被り、人格が裏口のまま。中立ポートを持てば「ACP シム」も「API 直叩き」も同じ面で選べる＝上位互換。よって「シムのみ・ポートなし」は却下、「ポート＋その下にシム or 直叩き」を採用。
- **legacy の raw-API 直叩きへ戻す（`legacy/app.py` 方式）**: ドメインが ACP 前提の同型性（ADR-0001の帰結）を失う。ポート無しの直叩きは不可。却下。ただし「API アダプタの中身」としては直叩きが正解＝ポートの背後に隔離するなら可。
- **MCP sampling を推論経路にする**: 逆トポロジ＋ deprecated（ADR-0001 で既に却下）。不採用を再確認。

## 影響 / 帰結
- `src/acp.py`: `AcpAgent` から `Agent`（Protocol/ABC）を抽出（`prompt/cancel/close`）。`ACPClient` は ACP アダプタの内部実装に留める。振る舞いは不変（既存テスト緑のまま）。
- **config**: バックエンド選択のつまみが要る（例 `ENGAWA_BACKEND=acp|api`、API 系は `base_url`/`model`）。ADR-0020 のモデル選択（住人=ANTHROPIC_MODEL / 客人=CODEX_CONFIG）は「ACP アダプタのモデル注入方法」と位置づけ直す＝API アダプタは自前の model 指定を持つ。
- **LM Studio 導入**: API アダプタ（`OpenAIAgent`→`localhost:1234/v1`）か ACP シムの2択（どちらもポートの1アダプタ）。
- **テスト**: ポート契約はモック実装でユニット化できる（現状 `Speaker` 注入と同じ土俵）。実 LLM の発話トーンは目視（ADR-0015/0025 と同じ分担）。
- **ADR-0021**（timeout/段階回復）は ACP アダプタ固有の関心事として整理。API アダプタは HTTP timeout/中断で別途担保。

## 備考（Open Questions）
- **認証差**: ACP はサブスク認証をエージェント任せ。API ローカルは無認証（localhost）。**クラウド API キーは ADR-0002 で依然禁止**＝API アダプタはローカル/自ホスト限定を既定線とする。
- **茶々にツール（MCP）を持たせるか**は ADR-0008（人格にツールを持たせない＝ファイルを触る誘惑を断つ）と要調整＝本 ADR とは別判断。ここでは「載せる口の場所」だけ確定し、載せる是非は保留。
- **cancel の意味マップ**: ACP=`session/cancel`、API=HTTP ストリーム中断。ポート契約は「進行中の prompt を速やかに畳む」意味だけを規定し、手段はアダプタ内。
- **長命セッション**（ADR-0005）: ACP は session を持つが、素の補完 API はステートレス＝API アダプタは会話履歴を自前で保持する必要がある（茶々の「地続きの文脈」を壊さないための実装責務）。
