# ADR-0012: 天気は実天気が真実、箱庭はそれに従属する（状態は実天気・手触りは箱庭）

- ステータス: Accepted
- 日付: 2026-06-27
- 関連: ADR-0011, ADR-0004, TECH_RULES §6, Backlog（前ティック差分 / 体感ナレーション層）

## 背景 / 課題
箱庭アーク（ADR-0011）で「雨の起承転結」のような天気フレーバーの物語を出したい。だが天気は実環境（Open-Meteo の大阪実天気）と連動する建前。脚本で「今から雨→雷→晴れ」を注入すると、実天気が晴れの時に矛盾する。
茶々は文脈の矛盾に気づく（P3実機検証で、こちらが与えた「夕日」に対し「今まだ朝の10時すぎちゃう？夕日には早い」と自分でツッコんだ）。捏造した天気はペルソナと没入を壊す。

## 決定
天気を **箱庭アークにしない**。イベント源を二層に分離する。
- **実天気（状態）= 読み取り専用の真実。** その「起承転結」は現実が描く。実装は **前ティック差分** で遷移（晴→曇→雨→上がり）を検出してナレーション化する（Backlog の体感ナレーション層がこれに当たる）。天気状態を発明しない。
- **箱庭アーク = 天気以外の出来事**（生き物・物音・人の気配）。各フェーズで実天気を **ゲート＆参照** し、矛盾しない（雀は「雨でない」をゲートに、降り出したら中断／反応）。
- 線引き：**状態は実天気、瞬間の手触りは箱庭。** State（晴雨・気温）は捏造禁止。Momentary texture（遠雷の一鳴り・一陣の風・軒の雨だれの音）は実状態の範囲内なら箱庭で可（粗いスナップショットでは否定できない瞬間の機微）。例：遠雷は実天気が雷コード(95/96/99)なら確実、ただの雨なら遠い一鳴りは許容、晴天時は禁止。
- **実天気の遷移が箱庭アークのトリガーになり得る**（実際に雨が上がったら→虹／雀が軒先から戻る／濡れた板の匂い）。**実天気が指揮者、箱庭が演奏。** 骨は現実、皮が箱庭。

## 検討した代替案
- **天気も脚本アーク化**: 実天気と矛盾し茶々が破綻に気づく。却下。
- **天気を実環境報告のみにし箱庭から完全分離（連動なし）**: 安全だが「天気フレーバーのアーク」が作れず物足りない。却下。トリガー連動で旨味を残す。
- **実天気を捨てて全部箱庭（疑似天気）**: 制御は楽だが「実環境に反応する住人」（ADR-0004）の核を捨てる。却下。

## 影響 / 帰結
- 天気アークは現実が降らせた時だけ起き、現実が上げた時だけ終わる＝レアで制御不能だが本物。捏造ゼロ。
- フィール確認は実演可能な生き物アーク（雀 等）で行う。天気側の前ティック差分ロジックはハーネスに偽の前ティック天気を食わせて素振り（テスト専用と明示）、本番は現実待ち。
- 「箱庭は実環境に従属」が、コードのゲート条件として明文化される。

## 備考
- これは ADR-0011（アークモデル）に対する制約。アークは自由だが、天気を扱うアークだけは実天気に縛られる。
- 大阪固定座標の利用者変更（Backlog）が入ると、ゲート／参照の対象も座標連動になる。→ **2026-07-11 実装済み**: 観測地点を config 化（`ENGAWA_WEATHER_LAT/_LON/_WEATHER_TZ`＋地名 `ENGAWA_PLACE_LABEL`・`engawa.json[weather]`・既定は大阪で現行維持）。天気の「真実」性はそのまま＝住む土地の実天気に従属する構造は不変で、対象座標が可変になっただけ。
