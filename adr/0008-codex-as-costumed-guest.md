# ADR-0008: Codex は「役を着せて呼ぶ客人」にする（対等な対話者にしない）

- ステータス: Accepted（P4で実装予定）
- 日付: 2026-06-26
- 関連: ADR-0001, ADR-0004, ADR-0007

## 背景 / 課題
ACPで Codex(GPT) も同型に繋がる（ADR-0001）。だが対等な会話相手にすると、避けたはずの「AI同士会話の不自然さ」（ADR-0004）が再燃する。

## 決定
Codexを環境イベントの一種「**来訪**」に格下げする。
- **器に役を着せて `/codex <人格>` で直接召喚**（茶々の取り次ぎを挟まない＝即・確実）。
- 客人セッションは来訪ごとに **使い捨て**（session/new → 数往復 → 破棄）。
- 滞在は **往復数で上限**（無限ループの安全弁）。
- 人格は **召喚時に prompt/system へ動的注入**（CLAUDE.md でなく）。
- 客人の出力は **ナレーション化**して茶々に渡す。

## 検討した代替案
- **固定の隣人（常駐・同一人格）**: 着せ替えの自由度なし。却下。
- **茶々が取り次ぐ（空振りあり）**: 世界観は濃いが、操作の気持ちよさを優先し **直接召喚** に決定。
- **自発来訪のみ**: ユーザー主導性が無い。**自発（時刻/確率/クールダウン）＋召喚** の2系統に。

## 影響 / 帰結
- マルチベンダーの旨味（Codexが繋がる）を取りつつ、機械臭さを回避。
- 来訪フロー（到来→滞在→辞去）を自発/召喚で共通化。P4で実装予定。

## 備考
- `/codex <自由テキスト>` はユーザー入力が客人の system に直入りする＝**配布時はプロンプトインジェクションの入口**（spec Open Question）。本人専用のV1は許容。
