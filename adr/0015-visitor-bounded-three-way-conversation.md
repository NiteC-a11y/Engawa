# ADR-0015: 環境イベントに「相互作用モード」を持たせ、客人(visitor)に *人間アンカーで有界な* 3人会話を解禁する

- ステータス: Accepted（方向確定・実装は別途。ADR-0008 の「一方向」割り切りを部分的に Supersede）
- 日付: 2026-06-27
- 関連: ADR-0004（自律AI↔AI会話からの転換）, ADR-0006（cancel優先）, ADR-0008（客人=役を着た有界の来訪）, ADR-0011（イベント=アーク）, 原則#3

## 背景 / 課題
客人(codex)は十分な対話能力を持つのに、ADR-0008 の v1 では台本3ビートの**一方向**＝宝の持ち腐れ。一方 ADR-0004 で「AI同士が自律的に延々喋る」不自然さは避けた経緯がある。

ここで「自然な3人の縁側」を考えると、登場人物は **私(人間)・茶々(住人)・客人(codex)** の3者で、対話の組み合わせは **私↔茶々 / 私↔客人 / 茶々↔客人 / 3人同時** の全てが成立しないと不自然（一部だけ通ると途端に「機能」臭くなる）。問題は **茶々↔客人＝AI↔AI** が ADR-0004 の地雷に見えること。

## 決定
1. **環境イベントに種別＋相互作用モードを持たせる**:
   - `ambient`（天気・時刻・箱庭アーク）= 主体性なし・**一方向**（茶々が反応するだけ）。
   - `visitor`（客人 codex）= 対話エージェント・**双方向（3人会話）可**。
2. **visitor 種別に「人間アンカーで有界な3人会話」を解禁**する。取り次ぎ（茶々が中継）ではなく、**全員が全員の発言を聞く「部屋」**にする。宛先（「小池さん、…」→客人／名前なし→文脈で茶々 or 両方）で誰が応えるかが自然に決まり、4つの組み合わせは別モードでなく**共有の場＋宛先から立ち上がる**。
3. **ADR-0004 の線引きを再定義**: 禁じるのは「AI↔AI そのもの」ではなく **「人間不在・無際限・機械的な自律往復」**。**人間が部屋に居て会話を駆動している限り**、茶々↔客人 の応酬は自然で安全。
4. **歯止め（決定の一部・ここが核）**:
   - **人間アンカー**: 3人会話は来訪中＝人間が関与している間だけ。背景タイマーで AI↔AI を回さない。
   - **連続 AI ターン上限**: 人間の発話の後、AI の応答は最大1〜2手で止めて人間待ち（機械的ピンポン化を防ぐ）。
   - **来訪は有界のまま**: ADR-0008 の核（常駐させない・使い捨て・滞在上限・cooldown）は維持。解禁するのは「一方向」割り切りだけ。

## 検討した代替案
- **取り次ぎ（茶々が客人に中継）**: 直接対話でなく不自然。ユーザーが明確に却下。
- **一方向のまま（v1）**: codex の対話能力を捨てる＝勿体ない。
- **上限なしの自律3者会話**: ADR-0004 の「水槽」に逆戻り。却下（歯止め必須）。

## 影響 / 帰結
- **構造変更**: 共有 transcript を **茶々にも客人にも**渡す（今の「客人→茶々ナレーション一方向」から双方向へ）。長命な茶々セッションと使い捨て客人セッションの**文脈共有**の設計が要る。
- **ターン管理がこの企画の最難関**: 「誰が次に喋るか／いつ止めて人間待ちにするか」を人間主導＋上限で詰めるのが全て。ここを雑にやると ADR-0004 に逆戻りするのが最大リスク。
- ADR-0008 は核（有界・非常駐）を残し、「一方向」だけ本 ADR で見直し。実装は別途（本 ADR は方向の確定）。

## 備考（実装時に詰める Open Questions）
