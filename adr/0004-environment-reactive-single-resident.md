# ADR-0004: 環境反応型の単体住人にする（マルチエージェント会話からの転換）

- ステータス: Accepted
- 日付: 2026-06-26
- 関連: ADR-0008

## 背景 / 課題
当初の構想は「Claude人格とGPT人格が自律会話するDiscord風アプリ」。だが、
1. AI同士の自律会話は機械臭さとの戦いになる。
2. 自律ループは課金・ToS的に締められる側（ADR-0002）。

## 決定
コンセプトを **「環境イベントに反応する単体の住人（茶々）」** に転換する。発話のトリガーを環境イベント（時刻・天気など）とし、住人はV1では **1体に限定**する。

## 検討した代替案
- **複数住人で会話させる**: AI同士の会話の不自然さが再燃。V1では却下（物足りなくなってから検討）。
- **常時自律ループ**: 機械臭さ＋課金リスク。却下。イベント駆動の疎な発話にする。

## 影響 / 帰結
- ターン制御を人工的に作らずに済む（環境イベントが発話の起点）。
- 発話が疎で課金的にもおとなしい。
- 名前「縁側(Engawa)」と完全に整合。spec全体がこの像に再構成された（v2）。

## 備考
- 客人（ADR-0008）は「会話相手」でなく環境イベントの一種として入れることで、この決定を壊さずにマルチベンダー(Codex)を取り込む。
