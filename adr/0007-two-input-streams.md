# ADR-0007: 入力を2系統に分ける（話しかけ / スラッシュ操作）

- ステータス: Accepted
- 日付: 2026-06-26
- 関連: ADR-0006, ADR-0008, spec v2 §8

## 背景 / 課題
ユーザー入力には「茶々への発話」と「縁側そのものへの操作（/codex 等）」の2種が混在する。

## 決定
- **通常テキスト** = 茶々への話しかけ（session/prompt へ）。
- **スラッシュ始まり** = 縁側への操作（コマンドディスパッチャ、茶々宛でない）。

## 影響 / 帰結
- Discord風UIと整合。
- ディスパッチャだけ拡張可能にし、V1は /codex（ADR-0008, P4）・/help・/quit のみ。将来 /weather /sleep /quiet 等。

## 備考
- なし。
