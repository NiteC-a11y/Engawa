# ADR-0010: ドット絵は差し替え可能なアセット層として分離する

- ステータス: Accepted
- 日付: 2026-06-26
- 関連: ADR-0009, spec v2 §14(P5)

## 背景 / 課題
茶々の見た目をドット絵（たまごっち風）にしたい。窓枠/レイアウトとキャラ絵は別物。

## 決定
- 窓枠・レイアウト・state機構は **コード側**。
- 茶々のスプライトは **別アセット層**。デモ段は procedural（コード図形）で動き・state切替の骨を作り、本番は **Aseprite製スプライトシートに差し替える**（`drawChacha()` をシート切り出しに置換）。
- 骨は再利用、皮を貼り替える構造。

## 検討した代替案
- **Pencilでキャラまで作る**: Pencilはコンポーネント/レイアウトUI向けで、ピクセルアートのキャラ作成は守備範囲外。窓枠には可（ただしキットは切る）。キャラには使わない。
- **たまごっち / Clawd 等の既存キャラに寄せる**: 商標/著作権アウト。完全オリジナルで作る。

## 影響 / 帰結
- アニメ=パラパラ（コマ送り）、移動=座標/transform、のハイブリッド。
- state（天気/時刻/気分）→ アニメ選択。imageSmoothing 無効（pixelated）必須。
- Aseprite（pixel-plugin 経由で Claude Code から生成可）の JSON でコマ/タイミングも管理。

## 備考
- UI配線は P3（双方向）の後。先に皮を被せると「話しかけられない水槽」になるため（順序は spec Phase Gate）。
